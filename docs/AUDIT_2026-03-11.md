## 1. SECURITY (Fix Immediately)

### 1.1 API Routes Missing Auth

| Route | Issue | Severity |
|-------|-------|----------|
| `/api/upload/route.ts` | No auth check - anyone can upload to fal.ai | CRITICAL |
| `/api/save-local/route.ts` | No auth + path traversal via `outputFolder` param | CRITICAL |
| `/api/fal/proxy/route.ts` | No auth check - proxies fal.ai API | HIGH |

**Path traversal detail:** `/api/save-local` accepts an `outputFolder` parameter used directly in `join(outputFolder, fileName)`. An attacker could write to `../../../etc/` (mitigated on Vercel by read-only filesystem, but still a code-level vulnerability).

### 1.2 Storage Bucket Wide Open

The `generations` storage bucket:
- **Upload policy**: `{public}` role, only checks `bucket_id = 'generations'`. No user ownership.
- **Delete policy**: Same - any user can delete any file.
- **No `file_size_limit`** (currently `null`) - unlimited upload size
- **No `allowed_mime_types`** (currently `null`) - any file type accepted

**Fix:**
- Change policies from `{public}` to `{authenticated}`
- Add path ownership check (path must start with `auth.uid()`)
- Set `file_size_limit` to 100MB
- Set `allowed_mime_types` to `['image/png', 'image/jpeg', 'image/webp', 'video/mp4']`

### 1.3 Webhook Endpoints - No Signature Verification

| Endpoint | Issue |
|----------|-------|
| `/api/webhooks/fal-completion` | No signature verification |
| `/api/webhooks/kie-completion` | No signature verification |

An attacker could forge webhooks to mark generations as complete with arbitrary URLs or trigger credit refunds.

**Note:** fal.ai doesn't provide webhook signatures. Mitigation: the `request_id` is verified against our DB (already done). Consider adding an HMAC secret as a query parameter for additional security.

### 1.4 Ghost Database Functions (Security Definer)

Two `SECURITY DEFINER` functions reference non-existent tables:
- `count_user_signatures(uuid)` - references `public.signatures` (doesn't exist)
- `count_user_email_signatures(uuid)` - references `public.email_signatures` (doesn't exist)

Both lack `search_path` setting. Leftover from another project sharing this Supabase instance.

**Fix:** `DROP FUNCTION count_user_signatures(uuid); DROP FUNCTION count_user_email_signatures(uuid);`

### 1.5 Leaked Password Protection Disabled

Supabase advisor reports leaked password protection is disabled in Auth settings.

**Fix:** Enable in Supabase dashboard > Auth > Settings.

### 1.6 Missing Security Headers

`next.config.ts` has no `async headers()` config. Production should have:
- `X-Frame-Options: DENY`
- `X-Content-Type-Options: nosniff`
- `Referrer-Policy: strict-origin-when-cross-origin`
- `Strict-Transport-Security` (HSTS)

### 1.7 Dead API Routes Still Accessible

| Route | Issue |
|-------|-------|
| `/api/credits/deduct` | Exposed POST endpoint, never called from frontend. Could be abused. |
| `/api/save-local` | Filesystem write endpoint - should not exist in production |
| `/api/backfill-metadata` | One-off migration endpoint, should be removed |
| `/api/migrate-generations` | One-off migration endpoint, should be removed |
| `/api/generate/poll` | Old polling route, replaced by `/api/generation-poll`. Never called. |

## 2. BUGS (Fix Soon)

### 2.1 credit_transactions.generation_id is ALWAYS NULL

All 152 credit transactions have `generation_id = NULL`. The FK constraint exists but the app code never passes it. This means:
- `refund_credits()` can never find the original deduction by generation_id - always falls back to "refund to topup pool"
- No audit trail linking credit deductions to specific generations
- The `idx_credit_tx_generation` index is useless

**Fix:** Pass `generationId` to `deductCredits()` in `/api/generate` and `/api/animate-shot` after the DB insert.

### 2.2 Fire-and-Forget Supabase Operations in useGenerations.ts

Three operations are not `await`ed:

```
Line ~95:  supabase.from("generations").update({ favorited: ... }).eq("id", id)  // NOT AWAITED
Line ~109: supabase.from("generations").delete().eq("id", id)                     // NOT AWAITED
Line ~131: supabase.from("generations").delete().in("id", ids)                    // NOT AWAITED
```

If the component unmounts before these complete, favorites won't persist and deletes won't execute.

### 2.3 useCredits.ts / usePricing.ts - Silent Failure

Both hooks return without updating state if fetch fails. Users see stale/undefined data with no error feedback.

### 2.4 generate-shot Still Uses Blocking fal.subscribe()

`/api/generate-shot/route.ts` uses `fal.subscribe(endpoint, { input, logs: true })` - the old synchronous pattern that blocks for 10-30+ seconds.

Both `/api/generate` and `/api/animate-shot` already use the queue pattern. `generate-shot` is the last holdout.

### 2.5 Missing try-catch in billing/portal and account DELETE

Both handlers have no try-catch wrapping - unhandled exceptions crash the request.

### 2.6 2 Auth Users Without Profiles

8 users in `auth.users` but only 6 profiles. Two users (`alexander.m.antonov@gmail.com`, `fam.executive@gmail.com`) have orphaned `signup_bonus` credit_transactions (100 credits each) but no profile row. The `handle_new_user` trigger failed silently for these users.

### 2.7 4 Generations Stored Display Name Instead of model_id

4 generations have `model_id` set to display names (`Grok Imagine`, `NanoBanana 2`) instead of proper IDs (`grok-imagine`, `nano-banana-2`). This was a client-side bug - breaks cost attribution and analytics.

### 2.8 Missing placeholder.svg

`src/components/generate/GalleryGrid.tsx:240` references `src="/placeholder.svg"` as a fallback. This file does NOT exist in `public/`. Will show a broken image.

### 2.9 experimental.optimizeCss Enabled Without critters Package

`next.config.ts` has `experimental.optimizeCss: true` but the required `critters` package is not installed. The setting silently does nothing.

## 3. DATABASE ISSUES

### 3.1 generations Missing updated_at Column

Every other mutable table has `updated_at` + trigger. `generations` does not, despite being updated frequently (url set on completion, favorited toggled).

### 3.2 profiles.plan AND profiles.subscription_tier - Redundant

Both columns exist, both default to `'free'`, all 6 profiles have identical values. Drop `plan`, keep `subscription_tier`.

### 3.3 RLS Policies Missing (select auth.uid()) Wrapper

16 policies across all tables use `auth.uid()` directly instead of `(select auth.uid())`. This causes per-row re-evaluation instead of once per query.

### 3.4 model_pricing Has Redundant Overlapping Policies

"Service role manages model pricing" policy covers ALL (including SELECT) but "Anyone can read" also covers SELECT. Redundant evaluation.

### 3.5 Dead Functions

- `handle_updated_at()` - identical to `update_updated_at()`, not used by any trigger
- `count_user_signatures()` - references non-existent table (covered in 1.4)
- `count_user_email_signatures()` - same

### 3.6 Inconsistent UUID Generation

Some tables use `uuid_generate_v4()` (requires extension), others use `gen_random_uuid()` (native). Standardize on `gen_random_uuid()`.

### 3.7 scenes.id Default is Misleading

Column is `text` with default `uuid_generate_v4()`, but actual values are `scene_1772837653500`. The default is never used.

### 3.8 Missing CHECK Constraints

- `credit_transactions.type` - should be `CHECK (type IN ('deduction', 'refund', 'signup_bonus', 'topup', 'adjustment'))`
- `credit_transactions.pool` - should be `CHECK (pool IN ('subscription', 'topup'))`

### 3.9 Unused Indexes

| Index | Table | Scans | Action |
|-------|-------|-------|--------|
| `idx_profiles_username` | profiles | 0 | DROP (duplicate of unique constraint) |
| `idx_generations_favorited` | generations | 0 | DROP until feature used |
| `idx_scenes_project` | scenes | 0 | DROP until projects active |
| `idx_projects_user` | projects | 0 | DROP until projects active |

### 3.10 profile-photos Bucket Likely Unused

Both `avatars` and `profile-photos` buckets exist. `avatars` has proper policies and 1 file. `profile-photos` has 0 files.

### 3.11 Always-NULL Columns on generations

| Column | Non-null (of 293) | Assessment |
|--------|-------------------|------------|
| `negative_prompt` | 0 | Dead unless non-FLUX models added |
| `seed` | 0 | Bug - should be populated from API response |
| `source_audio_url` | 0 | Keep if audio-to-video planned |
| `cost_estimate` | 1 | Nearly dead |
| `project_id` | 0 | Projects feature unused |
| `scene_id` | 0 | Generations not linked to scenes |

### 3.12 Credit Balance Discrepancy

Alexander's `profiles.credits_topup` = 997,838 but sum of his `credit_transactions` = -2,061 (net). ~999,899 credits were added via direct SQL UPDATE without a corresponding transaction record. This breaks the audit trail. All other users reconcile perfectly.
